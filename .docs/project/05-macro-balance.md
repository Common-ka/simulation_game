# Математическая спецификация экономики и гибридной монетизации браузерного Idle-тайкуна

Разработка экономической модели для браузерного Idle-тайкуна, сфокусированного на механике анпакинга посылок с маркетплейсов, требует филигранной настройки математических балансов. Специфика браузерных игровых сессий диктует необходимость учета сверхбыстрых таймингов: игрок должен испытывать непрерывное чувство микро-прогрессирования, которое периодически прерывается жесткими, математически обоснованными **«экономическими стенами»** (точками торможения).

Данный документ представляет собой актуальную техническую и математическую спецификацию всех базовых циклов (Core Loop), мета-систем, механик коллекционирования и гибридной монетизации. Значения синхронизированы с данными генераторов `generate_economy.py`, `recalculate_ips.py` и JSON-базами в папке `/Economy`.

## 1. Прогрессия тематических категорий и баланс сложности

Прогрессия базируется на последовательном открытии десяти тематических категорий товаров, начиная от базового «Барахла» и заканчивая премиальным «Гос. конфискатом».

> **Архитектурное примечание:** Цены открытия категорий хранятся в `Economy/Categories.json` и читаются симулятором динамически. Никаких захардкоженных массивов цен в C# нет.

### Фундаментальная формула стоимости
```math
P_n = P_0 \times K^n
```

Где:
- $P_n$ — стоимость покупки лота на $n$-м шаге.
- $P_0$ — начальная цена первого лота в категории.
- $K$ — коэффициент экспоненциального роста (диапазон **1.15** до **1.20**).

## 2. Математика лотов, анпакинга и пассивного дохода (IPS)

### Матрица редкости и вероятностей (Drop Rates)

| Редкость | Вероятность | Эффект при установке на полку |
| :--- | :--- | :--- |
| **Common** | 55.0% | Продать сразу (нет слота) |
| **Uncommon** | 25.0% | Продать сразу (нет слота) |
| **Rare** | 12.0% | `Flat_IPS` или `Mult_MPC` |
| **Epic** | 6.0% | `Flat_IPS` или `Mult_MPC` |
| **Legendary** | 1.8% | `Flat_IPS` или `Mult_MPC` |
| **Unique** | 0.2% | `Flat_IPS` или `Mult_MPC` |

> Предметы редкости **Rare и выше** дублируются в базе: каждый существует в варианте `Flat_IPS` и `Mult_MPC` с шансом 50/50. Это гарантирует, что полка всегда содержит оба типа бустов.

### Формула Time-Based IPS (актуальная)

Базовый доход `Flat_IPS` для `Unique`-предмета категории N вычисляется скриптом `recalculate_ips.py`:
```
Unique_IPS(N) = UnlockCost(N+1) / (5 * TargetTime)
```
Где:
- `5` — количество слотов на полке
- `TargetTime` = **64,800 сек (18 ч)** для категорий 0–5
- `TargetTime` = **907,200 сек (10.5 дней)** для категорий 6–8

Минимальный пол: `Unique_IPS` не менее **150**, `Rare_IPS` не менее **1.0**.

### Деградация IPS по редкости

| Редкость | Коэффициент от Unique_IPS |
| :--- | :--- |
| **Rare** | / 150 |
| **Epic** | / 30 |
| **Legendary** | / 6 |
| **Unique** | × 1.0 |

### Итоговая формула дохода (аддитивная)

```math
CurrentIPS = AdjustedBaseIPS \times (1.0 + PrestigeBonus + BlackMarketBonus)
```

Где:
- `AdjustedBaseIPS` — сумма `Flat_IPS` всех предметов на полке, с применением `×1.2` к предметам из завершённых сетов стикеров
- `PrestigeBonus = PermanentPrestigeMultiplier - 1.0`
- `BlackMarketBonus = min(Сумма EffectValue артефактов +Total_IPS% / 100, HardCap)`
- `HardCap = BaseHardCapPercent / 100 = 3.0` (т.е. максимум +300% от BM)

### Формула силы клика

```math
MPC = BaseClickPower + ManualTapPower + (CurrentIPS \times 0.1)
```

## 4. Gacha-система и Альбом стикеров

Коллекционирование стикеров — мета-система, сохраняющая прогресс после **Престижа**. Стикеры и сеты **не обнуляются** при Престиже; **Звёздная пыль** обнуляется и конвертируется в Prestige Points.

### Механика выпадения

- Шанс выпадения стикера при открытии любой коробки: **5%**
- Стикер привязан к конкретному предмету (по ID из `LootTable.json`)
- Дубликат → конвертируется в **Звёздную пыль** по таблице `DustBurnTable`

### Таблица Звёздной пыли (DustBurnTable)

| Редкость | Пыль за дубликат | Стоимость крафта |
| :--- | :--- | :--- |
| Common | 1 | 5 |
| Uncommon | 3 | 15 |
| Rare | 10 | 50 |
| Epic | 50 | 250 |
| Legendary | 250 | 1,250 |
| Unique | 1,000 | 5,000 |
| Black_Market_Exclusive | 2,000 | 10,000 |

### Авто-крафт

Симулятор автоматически тратит накопленную пыль на покупку недостающих стикеров. Приоритет: от дешёвых редкостей (Common) к дорогим (Unique) в рамках открытых категорий. 1 крафт за такт симуляции.

### Синергии (Бонусы завершённых сетов)

- **Обычный сет (одна категория):** `×1.2` к `Flat_IPS` всех предметов этой категории на полке
- **Сет Чёрного рынка:** `×1.05` (зарезервировано)

При завершении сета в лог: `Собран сет категории [Имя]! Применен множитель 1.2x.`

### Pity-система (защита от неудач)

- Ожидаемое число коробок на сет из **11 уникальных предметов** при шансе 5%: ~228 коробок
- Зарезервирован Hard Pity Threshold: 250 коробок

## 5. Черный рынок (Black Market)

Активируется **Ключом**. Ключ можно получить:
- **Туториал:** 1 ключ бесплатно при старте Эпохи 1 → хардкодный дроп артефакта `+Total_IPS% (ID 1005)`
- **Случайный дроп:** шанс **0.05%** за коробку с кулдауном **2 часа** между дропами
- **IAP:** Master Key Pack

### Артефакты Чёрного рынка

Типы эффектов: `+Total_IPS%`, `+Gacha_Chance%`, `-Shop_Prices%`, `+Offline_Time`.
База из 50 артефактов в `Economy/BlackMarketArtifacts.json`.

### Лимит бонуса IPS от артефактов

```math
BlackMarketBonus = min\left(\frac{\sum EffectValue_{+Total\_IPS\%}}{100},\ 3.0 \times (1 + 0.1 \times PrestigeCount)\right)
```

Базовый хардкап: **+300%** (т.е. максимум ×4 от базового IPS).

## 6. Механика Престижа

Добровольный сброс прогресса в обмен на **Prestige Points (PP)**.

### Что сбрасывается, что сохраняется

| Параметр | Сбрасывается | Сохраняется |
| :--- | :--- | :--- |
| SoftCurrency | ✅ | — |
| Апгрейды (авто/ручной клик) | ✅ | — |
| Полка (предметы) | ✅ | — |
| Звёздная пыль | ✅ (конвертируется) | — |
| Артефакты Чёрного рынка | — | ✅ |
| Разблокированные стикеры | — | ✅ |
| Завершённые сеты | — | ✅ |
| PermanentPrestigeMultiplier | — | ✅ (накапливается) |

### Формула начисления Prestige Points

```math
PP = \left(\frac{\text{Lifetime Earnings}}{1{,}000{,}000{,}000}\right)^{0.38} + \frac{\text{TotalStardustAtPrestige}}{10{,}000}
```

- Первое слагаемое — основная прогрессия от заработка.
- Второе слагаемое — вклад пыли: при 7М пыли даёт ~700 PP, что составляет ≤15% от типичного итога.

### Глобальный множитель дохода

```math
M_{pres} = 1 + (PP \times 0.02)
```

Каждое очко PP даёт **+2%** к доходу навсегда. Применяется аддитивно вместе с бонусом Чёрного рынка:

```math
TotalMultiplier = 1.0 + PrestigeBonus + BlackMarketBonus
```

### Логирование Престижа

```
[Prestige] Сконвертировано X пыли в Y очков престижа. Текущий запас пыли: 0.
Prestige Points (PP): A (доход) + B (пыль) = C -> Глобальный Множитель: Mx
```

## 7. Архитектура симуляции (MetaSimulationController)

Симуляция разбита на **Эпохи**, управляемые `MetaSimulationController.cs`:

1. **Эпоха 1** — 14 игровых дней, стартовый ключ ЧР.
2. **Триггер Престижа** — автоматически на границе 14 дней.
3. **Перенос данных** (`RetainedData`) — артефакты, стикеры, сеты, множитель.
4. **Эпоха 2** — 14 игровых дней с накопленными бонусами. Лог в отдельный файл `SimulationLog_Epoch2.txt`.

Цель Эпохи 2: зафиксировать тайминги открытия категорий 8, 9, 10 с учётом буста Престижа.

## 9. Гибридная монетизация

1. **Interstitial Ads**: При переходах (открытие новой категории, выход с Черного рынка). Кулдаун — 4–5 минут.
2. **Rewarded Video (RV)**:
   - **X-Ray сканер**: Просвечивание коробок (лимит 3/сессия).
   - **Продление таймера**: +2 мин на Черном рынке (FOMO-эффект).
   - **Мгновенная очистка склада**: Устранение рутины с бонусом 150%.
3. **In-App Purchases (IAP)**:
   - **Starter Pack ($1.99)**: 2 полки + 500 Гемов + Ультра-раритет.
   - **Ad-Free ($4.99)**: Отключение рекламы.
   - **Master Key Pack ($2.99)**: 5 ключей от Черного рынка.
