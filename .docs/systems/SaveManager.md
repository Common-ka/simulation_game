# 💾 SaveManager (Фолбэк)

Система отвечает за сохранение и загрузку состояния игрока. На данном этапе реализована упрощенная (фолбэк) версия для локального хранения через `Newtonsoft.Json` и `PlayerPrefs`. В будущем будет заменена/интегрирована с облачными сохранениями `Yandex Games SDK`.

## Ключевые классы
* `SaveManager`: Singleton (MonoBehaviour). При старте загружает данные, а затем автоматически вызывает автосохранение `Save()` каждые 30 секунд (через `InvokeRepeating`).
* `RetainedData`: Сериализуемый класс, в котором хранятся все важные метрики и инвентарь игрока.

## Публичное API
* `SaveManager.Instance` — статичный синглтон-доступ.
* `SaveManager.Instance.Data` — инстанс `RetainedData` (чтение/запись данных).
* `SaveManager.Instance.Save()` — принудительное сохранение текущего состояния.
* `SaveManager.Instance.Load()` — загрузка (происходит автоматически при `Awake()`).

## Важное
* Все данные сериализуются `Newtonsoft.Json` по ключу `"idle_fallback_save"` в `PlayerPrefs`. Если файла нет, инициализируется "чистый" экземпляр `RetainedData`.
* Сохраненные данные НЕ должны очищаться в обход логики престижа (только согласно `PrestigeService`).
