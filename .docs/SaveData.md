# Модель Сохранений (SaveData)
## Unclaimed Assets · Справочник разработчика

Документирует структуру, жизненный цикл и логику сброса сохраняемых данных. 
В Unity-проекте сохранения управляются через `PluginYG2`.

---

## 1. Архитектура и Интеграция

Данные хранятся в статическом поле плагина: `YG2.saves`.
Чтобы добавить новые поля, необходимо использовать `partial class SavesYG` в пространстве имён `YG`. Это позволяет не менять код самого плагина.

**Путь в проекте:** `Assets/Scripts/Core/Data/SavesExtension.cs`

> **Важно:** Не используйте `PlayerPrefs` или другие системы. Единый источник правды для стейта сессии (Session State) — это всегда `YG2.saves`.

---

## 2. Полная структура данных

Ниже приведена полная структура `SavesYG` с типами и значениями по умолчанию. **Любое изменение этой структуры должно сопровождаться обновлением версии `EconomyVersion`.**

```csharp
using YG;
using System.Collections.Generic;

namespace YG {
    public partial class SavesYG {
        // ==========================================
        // 1. МЕТАДАННЫЕ И СИСТЕМНОЕ
        // ==========================================
        public string   EconomyVersion            = "1.0";       // Текущая схема данных
        public long     LastOnlineTimestamp       = 0L;          // Время выхода (Unix time utc)
        public bool     IsTutorialComplete        = false;
        public bool     IsTutorialKeyGiven        = false;       // Выдан ли стартовый ключ

        // ==========================================
        // 2. ВАЛЮТЫ И ЭКОНОМИКА
        // ==========================================
        public double   SoftCurrency              = 0.0;         // Деньги из контейнеров
        public double   CurrencyStardust          = 0.0;         // Пыль (Гача)
        public int      HardCurrency              = 0;           // ЯМ (Яндекс Монеты)
        public int      BlackMarketKeys           = 0;           // Золотые ключи (ЧР)

        // ==========================================
        // 3. ПРОГРЕСС И ПРЕСТИЖ
        // ==========================================
        public int      UnlockedCategoryIndex     = 0;           // 0..9 (Барахло .. Гос. Конфискат)
        public double   LifetimeEarnings          = 0.0;         // Всего заработано (для лидерборда/аналитики)
        public int      PrestigeCount             = 0;           // Количество "Обновлений контракта"
        public double   PermanentPrestigeMultiplier = 1.0;       // Текущий множитель престижа

        // ==========================================
        // 4. КОЛЛЕКЦИИ (Постоянные)
        // ==========================================
        public List<int>    UnlockedStickerIDs    = new List<int>();
        public List<string> CompletedSetNames     = new List<string>();
        
        // ==========================================
        // 5. ИНВЕНТАРЬ И ПОЛКИ (Сбрасываемые)
        // ==========================================
        public List<int>    ShelfItemIDs          = new List<int>(); // ID предметов на витрине (-1 = пустой слот)
        
        // ==========================================
        // 6. ЧЁРНЫЙ РЫНОК
        // ==========================================
        public List<int>    OwnedArtifactIDs      = new List<int>(); // Постоянные бонусы ЧР
        public long         LastKeyDropTimestamp  = 0L;              // Для ограничения дропа ключей

        // ==========================================
        // 7. УЛУЧШЕНИЯ (Сбрасываемые)
        // ==========================================
        // Словарь: "UpgradeID" -> Уровень апгрейда
        // PluginYG поддерживает сериализацию Dictionary<string, int>, но проще использовать параллельные List или классы, если Dictionary сбоит.
        public Dictionary<string, int> UpgradeLevels = new Dictionary<string, int>();

        // ==========================================
        // 8. МОНЕТИЗАЦИЯ (VIP)
        // ==========================================
        public bool     IsVipSilver               = false;
        public bool     IsVipGold                 = false;
        public long     VipExpiryTimestamp        = 0L;          // Дата окончания подписки
    }
}
```

## 3. Логика Сброса (Престиж / Обновление контракта)

При выполнении Престижа («Обновление контракта»), игрок получает перманентный множитель, но часть прогресса сбрасывается к базовым значениям. Это реализуется в методе `PrestigeService.ApplyPrestige`.

### Что **СБРАСЫВАЕТСЯ** (Обнуляется):
- `SoftCurrency` = 0
- `UnlockedCategoryIndex` = 0 (Откат до Категории 1: Барахло)
- `ShelfItemIDs` = Пустые слоты (-1)
- `UpgradeLevels` = Очищается (все улучшения становятся 0 уровня)

### Что **СОХРАНЯЕТСЯ** (Не трогаем):
- `HardCurrency` (ЯМ)
- `CurrencyStardust` (Звёздная пыль)
- `BlackMarketKeys` (Золотые ключи)
- `UnlockedStickerIDs` и `CompletedSetNames` (Гача и коллекции)
- `OwnedArtifactIDs` (Артефакты Чёрного рынка)
- `IsVipSilver`, `IsVipGold`, `VipExpiryTimestamp`
- Системные и анти-чит метки (`LastKeyDropTimestamp`)

### Что **ИЗМЕНЯЕТСЯ**:
- `PrestigeCount` += 1
- `PermanentPrestigeMultiplier` пересчитывается по формуле от текущего `LifetimeEarnings`

## 4. Версионирование и Миграции

Со временем `SavesYG` будет меняться (добавились новые поля, изменился тип данных). Яндекс Игры восстанавливают старую структуру. При загрузке игры необходимо проверять версию.

```csharp
// SaveManager.cs
public void CheckAndMigrateSaves() {
    string currentVersion = "1.1"; // Версия в коде
    
    if (YG2.saves.EconomyVersion == "1.0") {
        // Миграция 1.0 -> 1.1
        // Например: пыль раньше была int, стала double (или изменилось начисление)
        YG2.saves.CurrencyStardust += 100; // Бонус за миграцию
        YG2.saves.EconomyVersion = "1.1";
        YG2.SaveProgress();
    }
}
```

## 5. Валидация стейта (Защита от коррупции)

Иногда кеш браузера портится, или приходят кривые данные JSON. При загрузке (событие `YG2.onGetSDKData`) вызывается `SaveManager.ValidateState()`:

1. **Проверка коллекций:** Если `ShelfItemIDs` пуст (а должен содержать минимум столько элементов, сколько рабочих слотов на полке), массив пересоздается с длинами = количеству слотов (-1 как значения). 
2. **Анализ IPS:** Если `CalculateIPS()` выдает бесконечность или NaN, `SoftCurrency` сбрасывается к нулю (защита от переполнения double).
3. **VIP-подписки:** Очистка VIP-статуса, если `VipExpiryTimestamp < ServerTimeProvider.Current`.

---

## 6. Правила для агента при работе с SaveData

- **Запись:** Модифицировать стейт только через сервисы (например, `EconomyService`, `GachaService`). UI не должен писать напрямую в `YG2.saves.X`.
- **Сохранение:** Плагин ограничивает частоту сохранений. Вызывать `YG2.SaveProgress()` только по значимым триггерам:
    1. Покупка категории/апгрейда/артефакта.
    2. Выпадение важного лута (стикер, ключ).
    3. Открытие/закрытие игры `GameplayStart/Stop`.
    4. По таймеру авто-сейва (например, 1 раз в 60 секунд, но не чаще 1 раза в 10 секунд).
- **Сложные типы:** PluginYG может плохо сериализовать вложенные кастомные объекты и многомерные массивы. Используйте базовые структуры (List, Dictionary<primitive, primitive>, одномерные массивы).
