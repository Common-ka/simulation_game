# 1.4_SaveManager

## Контекст
Необходимо реализовать локальную систему сохранений `SaveManager.cs` в `Assets/Scripts/Core/`. В будущем игра будет использовать плагин Яндекса для сохранения, но на данном этапе требуется реализовать фолбэк-решение — локальное сохранение через сериализацию структуры `RetainedData` в JSON и запись в `PlayerPrefs`. Также необходимо реализовать автосохранение прогресса.

## Связанная документация (ОПТИМИЗАЦИЯ ТОКЕНОВ)
Вам нужно опираться строго на следующие документы:
- `.docs/project/10-save-data-architecture.md` (для понимания полей сохранения)
- `.docs/systems/INDEX.md` (для добавления записи в картотеку систем)

**СТРОГО ЗАПРЕЩЕНО** читать другие базовые документы из папки `.docs/project/`.

## Архитектурные требования
- **Отказ от Yandex SDK:** Временно абстрагироваться от плагина `YG2`.
- **Структура данных:** Создайте C#-класс/структуру `RetainedData`. Состав полей возьмите из пункта "Полная структура данных" документа `10-save-data-architecture.md` (SoftCurrency, UnlockedCategoryIndex и т.д.).
- **Механизм сохранения:** Использовать `JsonUtility` для парсинга и `PlayerPrefs.SetString()` / `PlayerPrefs.GetString()`.
- **Автосохранение:** Реализовать период таймера 30 секунд. Используйте `InvokeRepeating` или корутину, никаких тяжелых операций в `Update()`. Не хардкодьте `30f` непосредственно в коде исполнения — вынесите константу или `[SerializeField]`.
- **Обновление LTM (Длинная память):** Не забудьте обновить файл `INDEX.md`, добавив туда запись о новой системе `SaveManager`.

## Критерии приемки (Acceptance Criteria)
- [ ] Создана структура `RetainedData` со всеми актуальными полями, указанными в доке архитектуры (в качестве отдельного класса/скрипта или внутри SaveManager).
- [ ] Написан скрипт `Assets/Scripts/Core/SaveManager.cs`.
- [ ] Реализован метод `Save()` для сохранения состояния в JSON -> `PlayerPrefs`.
- [ ] Реализован метод `Load()` (создает новую сущность `RetainedData`, если сохраненных данных в `PlayerPrefs` нет).
- [ ] Настроено автосохранение раз в 30 секунд.
- [ ] Добавлена техническая информация и API `SaveManager` в `.docs/systems/INDEX.md` (как LTM-память).
