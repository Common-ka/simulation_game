# M3.3 — Main Screen Skeleton (Visual Only)

**Контекст:**
Цель — собрать единый визуальный каркас (Root UI) с плейсхолдерами для основных элементов экрана (HUD, витрина, кнопка Гачи и навигация) для того, чтобы увидеть всю композицию экрана. 
Это чистая вёрстка без C# логики, в дальнейшем к этим `#id` будут привязываться реальные скрипты и компоненты.

**Связанная документация (ОПТИМИЗАЦИЯ ТОКЕНОВ):**
- Основной референс зон: `.docs/project/09-ui-screens-guidelines.md` (Раздел 1 "Постоянный HUD" и Раздел 2 "Экран Геймплея").
- **Строго запрещено** тратить токены на чтение других базовых архитектурных документов для этой задачи.

**Архитектурные требования:**
1. Создать корневой документ окна: `Assets/Scripts/UI_Toolkit/UXML/MainScreen.uxml`.
2. Создать стили окна: `Assets/Scripts/UI_Toolkit/USS/MainScreen.uss` и подключить в него `Variables.uss` и `Common.uss` (созданные ранее).
3. Верстка выполняется только через **Flexbox** (относительное и абсолютное позиционирование, где уместно для шапки/подвала).
4. Обязательна разбивка на основные зоны (4 части):
   * **Шапка (HUD):** SoftCurrency, IPS, HardCurrency, Stardust, Keys.
   * **Геймплей:** Центральная витрина на 5 слотов (цветные квадраты-заглушки или пустые контейнеры с dummy текстом).
   * **Магазин-Геймплей:** Блок текущей категории (инфо-блок) и массивная кнопка "Крутить/Buy/Spin".
   * **Подвал (Футер):** 4 кнопки навигации (Upgrades, Gacha, Shop, Black Market).
5. **ID элементов:** Каждому логическому узлу, тексту, кнопке назначить уникальный `#id` (например: `#hud-soft-currency`, `#shelf-slot-1`, `#shop-spin-btn`, `#nav-btn-upgrades` и т.д.).
6. **Никакой C# логики.** Эта задача только про визуальную сборку (Mockup) структуры. 

**Критерии приемки (Acceptance Criteria):**
- [ ] Файл `MainScreen.uxml` создан и ссылается на `MainScreen.uss`, `Variables.uss`, `Common.uss`.
- [ ] В UXML четко выделены 4 секции: Header, Gameplay, Shop Info, Footer.
- [ ] При открытии UXML в UI Builder структура выглядит завершенной и не ломается (плейсхолдеры занимают свои места).
- [ ] Присутствуют dummy-тексты (например "1.5M") и все нужные элементы из дизайна.
- [ ] Все изменяемые элементы и кнопки имеют `#id`.
- [ ] C# классы и скрипты **НЕ СОЗДАВАЛИСЬ**.
- [ ] (При необходимости) Обновлён `INDEX.md` систем, если добавляется новая структура хранения UXML/USS, хотя для интерфейсов это опционально.
