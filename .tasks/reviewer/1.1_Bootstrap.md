# Задача 1.1 — Bootstrap: BootstrapController (Заглушки)

## Название
Bootstrap — Часть 1: BootstrapController с заглушками шагов и переключением на Game

## Контекст
Bootstrap.unity — это точка входа игры. Единственная задача сцены: выполнить инициализацию и перейти в `Game.unity`. На данном этапе Yandex SDK и SpriteSheetLoader не подключены. Все шаги, которые их требуют, реализуются как заглушки (stub) с выводом в консоль.

После выполнения этой задачи у проекта появится работающая последовательность запуска, и Агент-Воркер обязан задокументировать созданную систему в `INDEX.md`.

## Архитектурные требования

- Файл: `Assets/Scripts/Bootstrap/BootstrapController.cs`
- Тип: `MonoBehaviour`, прикреплён к единственному GameObject в `Bootstrap.unity`
- Паттерн инициализации: **корутина** (`IEnumerator Start()`), шаги выполняются последовательно с `yield return`
- Нет `Update()` — никакой логики в Update
- Каждый шаг логирует прогресс через `Debug.Log()`
- После всех шагов — `SceneManager.LoadScene("Game")`
- Никаких Magic Numbers: имя сцены выносится в `private const string`
- Не использовать `FindObjectOfType` / `GetComponent` внутри корутины (будущие сервисы будут создаваться через прямую ссылку или статику)

---

## Шаги инициализации (согласно 01-project-architecture.md)

Реализовать как `yield return null` (одна итерация) + `Debug.Log()` для каждого шага:

| # | Метод-заглушка | Лог в консоль |
|---|---|---|
| 1 | `LoadGameData()` | `"[Bootstrap] Loading GameData..."` |
| 2 | `LoadSaveData()` | `"[Bootstrap] Loading SaveData..."` |
| 3 | `PreloadSpriteSheet()` | `"[Bootstrap] Preloading SpriteSheet (stub)..."` |
| 4 | `InitYandexSDK()` | `"[Bootstrap] Yandex SDK Init (stub)..."` |

После шага 4: `Debug.Log("[Bootstrap] Done. Loading Game scene...")` и `SceneManager.LoadScene(GameSceneName)`.
