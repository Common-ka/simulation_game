# 1.4: SaveManager (Фолбэк-сохранение)

**Контекст:** 
Необходимо реализовать ядро системы сохранений, чтобы прогресс игрока (валюты, апгрейды) не терялся между сессиями. Поскольку плагин Yandex Games (PluginYG2) еще не проинициализирован и не интегрирован, на данном этапе требуется реализовать так называемое "фолбэк-сохранение" — временный механизм сериализации данных в JSON и их записи локально через `PlayerPrefs`. Это позволит тестировать механики с сохранением состояния уже сейчас.

**Связанная документация (ОПТИМИЗАЦИЯ ТОКЕНОВ):**
- `.docs/project/10-save-data-architecture.md` — Используй этот файл для понимания того, какие данные должны сохраняться (ориентируйся на структуру `SavesYG`, но адаптируй ее в локальный класс `RetainedData`).

**Архитектурные требования:**
- **Разделение слоев:** `SaveManager` не должен зависеть от визуального интерфейса (UI). Он отвечает исключительно за логику хранения данных. Обращайся к `AGENTS.md` - "Никакой логики отображения в Core".
- **Структура модели:** Создай класс(или структуру) `RetainedData`, который содержит ключевые поля из `10-save-data-architecture.md` (EconomyVersion, SoftCurrency, HardCurrency, UnlockedCategoryIndex, и т.д.).
- **Локальное Хранилище:** Использовать класс `JsonConvert` (из пакета `Newtonsoft.Json`) для сериализации в JSON. Это позволит без проблем сохранять словари и сложные структуры. Строку JSON сохранять в `PlayerPrefs` по заранее заданному ключу (например, `"idle_fallback_save"`).
- **Оптимизация:** Автосохранение должно работать каждые 30 секунд. Используй `InvokeRepeating` или корутину вместо тяжелой логики в `Update()`.

**Критерии приемки (Acceptance Criteria):**
- [ ] В папке `Assets/Scripts/Core/` создан скрипт `SaveManager.cs`.
- [ ] Создан сериализуемый класс `RetainedData` (внутри `SaveManager` или отдельным файлом `RetainedData.cs`) с полями, соответствующими базовой экономике проекта (опираясь на структуру из `10-save-data-architecture.md`).
- [ ] В `SaveManager` реализован метод `Load()`, который пытается десериализовать данные из `PlayerPrefs`. Если данных нет или JSON пуст, создается "чистый" экземпляр `RetainedData`.
- [ ] В `SaveManager` реализован метод `Save()`, который сохраняет текущий `RetainedData` в JSON и записывает в `PlayerPrefs`.
- [ ] Реализовано автосохранение прогресса один раз в 30 секунд.
- [ ] Добавлено базовое API (например, статичное свойство `Instance` или `Get()`) для доступа других подсистем к загруженным данным.
- [ ] **ОБЯЗАТЕЛЬНАЯ ДОКУМЕНТАЦИЯ:** Разработчик (Worker) должен актуализировать файл `.docs/systems/INDEX.md`, добавив туда новую LTM (Long-Term Memory) запись о системе `SaveManager`.
