# Название: GachaController (Открытие коробок)

Основная механика игры — покупка лотов (коробок) с предметами разной редкости. `GachaController` будет отвечать за математику выпадения (Gacha), списание валюты и маршрутизацию: ценный лут (Rare+) отправляется на полку (`ShelfManager`), а дешёвый (Common/Uncommon) попадает в инвентарь игрока, откуда может быть продан позже.

**Связанная документация (ОПТИМИЗАЦИЯ ТОКЕНОВ):**
- `.docs/project/05-macro-balance.md` (Раздел 2: Матрица редкости и вероятностей)
- Открывать другие базовые файлы **строго запрещено**. Опирайся на правила из `AGENTS.md`.

**Архитектурные требования:**
1. **Никаких Magic Numbers**: Вероятности (дропрейты) должны быть выровнены по таблице из документации. По возможности используй массив/структуру с вероятностями или `SerializedField`, а не хардкодь их глубоко в методах.
2. **Decoupled**: Контроллер не должен знать ничего про UI. Вся логика выполняется на бэке: списание валюты, генерация предмета, добавление его на полку.
3. Пространства имён: Используй соответствующие для проекта (например, `UnclaimedAssets.Gacha`).

**Критерии приемки (Acceptance Criteria):**
- [ ] Создать скрипт `GachaController.cs` (возможно как `Singleton` или `MonoBehaviour` в `Assets/Scripts/Gacha/`).
- [ ] Добавить в `GameManager.cs` методы `TrySpendCurrency(double amount)` и `AddCurrency(double amount)` (т.к. они сейчас там отсутствуют), не сломав начисление текущего `SoftCurrency` в тике игры.
- [ ] Создать (если не существуют) перечисление `ItemRarity` (Common, Uncommon, Rare, Epic, Legendary, Unique).
- [ ] Реализовать публичный метод `BuyLoot(int categoryIndex, double cost)`.
- [ ] Описать глобальное событие `public static event Action<int> OnLootBoxOpened;` (передающее `categoryIndex`), чтобы другие системы (Стикеры, Черный рынок) могли отслеживать открытие коробок.
- [ ] `BuyLoot` должен списывать `SoftCurrency` у `GameManager` перед генерацией лота. Если валюты недостаточно — прерывать операцию.
- [ ] Реализовать рандомизатор на базе шансов:
  - Common: 55.0%
  - Uncommon: 25.0%
  - Rare: 12.0%
  - Epic: 6.0%
  - Legendary: 1.8%
  - Unique: 0.2%
- [ ] В конце успешного выполнения `BuyLoot` (после всех списаний и зачислений) обязательно вызывай событие `OnLootBoxOpened?.Invoke(categoryIndex)`.
- [ ] Если выпадает Common или Uncommon: предмет не занимает место на полке, а добавляется в инвентарь (или логируется как добавленный, если `InventoryManager` еще не написан). При продаже такого предмета цена должна извлекаться из поля `SellPrice` в базе `Assets/Resources/GameData/LootTable.json` по ID полученного предмета.
- [ ] Если выпадает Rare, Epic, Legendary или Unique: попытаться отправить предмет в менеджер полок `ShelfManager.Instance.TryAddItem(...)` Только при условии наличия свободных слотов. (Если места нет — обработать это без автоматического удаления существующих предметов или ошибки).
- [ ] Проверить отсутствие ошибок компиляции в проекте Unity.
- [ ] Описать систему `GachaController` в файле `.docs/systems/INDEX.md` (для сохранения контекста ИИ-агентов).
