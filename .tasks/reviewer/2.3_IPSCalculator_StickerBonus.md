# Техническое задание: Исправление IPSCalculator - Бонус Стикеров (2.3)

**Название:** Учет синергии стикеров (Completed Sets) в математике IPS

**Контекст:** В предыдущей реализации `IPSCalculator.cs` мы сознательно опустили синергию от альбома стикеров. Пришло время исправить этот недочет (забытый Промптером) и добавить бонус ×1.2 к доходу предметов от полностью собранных сетов.

**Связанная документация (ОПТИМИЗАЦИЯ ТОКЕНОВ):**
- Геймдизайн: раздел "1.2 Формула текущего дохода" и "3.6 Бонус завершённого сета" в `.docs/project/03-technical-gdd.md`.
- Сохранения: структура `CompletedSetNames` в файле `.docs/project/10-save-data-architecture.md`.

**Архитектурные требования:**
1. Отредактировать файл `Assets/Scripts/Economy/IPSCalculator.cs`.
2. Изменить входные параметры функции. Теперь `GetCurrentIPS` должна получать информацию об открытых сетах. Рекомендуемая сигнатура:
   `public static double GetCurrentIPS(ShelfManager shelf, double prestigeMultiplier, double blackMarketBonus, List<string> completedSets)`
   (Внимание! Чтобы не нарушать архитектуру, класс не должен жестко зависеть от PluginYG, поэтому мы передаем список снаружи).
3. **Изменение логики:**
   - Во время суммирования `AdjustedBaseIPS`, проверьте каждый предмет с `EffectType == "Flat_IPS"`.
   - Если категория этого конкретного предмета (`Item.Category`) содержится в массиве `completedSets` — умножаем его индивидуальный `EffectValue` на 1.2.
   - Иначе используем базовое значение `EffectValue`.
4. В местах, где уже вызывается этот метод (например, `GameManager.cs`), добавить чтение `YG2.saves.CompletedSetNames` и передать его в качестве аргумента.

**Критерии приемки (Acceptance Criteria):**
- [x] Отредактирован метод `GetCurrentIPS` в `Assets/Scripts/Economy/IPSCalculator.cs` (и добавлена зависимость `using System.Collections.Generic;` если нужно).
- [x] К отдельным предметам из открытых сетов применяется множитель `×1.2`.
- [x] Добавлены изменения вызова метода в `GameManager.cs`, чтобы код компилировался (передан список коллекций `YG2.saves.CompletedSetNames`).
- [x] Итоговая математика не перемножает глобальные бонусы друг на друга.
- [x] Ошибки компиляции Unity отсутствуют.

## Замечания от Ревьювера
1. **Ошибка компиляции**: В скрипте `IPSCalculator.cs` вызывается `item.Category`, однако у структуры `ShelfItemData` (в файле `ShelfManager.cs`) нет поля `Category`. Отсюда возникает ошибка: `error CS1061: 'ShelfItemData' does not contain a definition for 'Category'`. Необходимо добавить строковое поле `Category` в `ShelfItemData` или пересмотреть логику.
2. **Игнорирование ТЗ (GameManager)**: В файле `Assets/Scripts/Core/GameManager.cs` метод `GetCurrentIPS` не вызывается. Сейчас там всё ещё хардкод `_softCurrency += 1;`. Соответственно, в него не передаётся `YG2.saves.CompletedSetNames`. Требование из ТЗ проигнорировано.

Задача отклонена, пожалуйста, исправьте эти баги и убедитесь, что проект компилируется.
