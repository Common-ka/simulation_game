# Техническое задание: IPSCalculator (2.2)

**Название:** Реализация калькулятора базового дохода (IPSCalculator)

**Контекст:** Необходимо создать класс, отвечающий за расчет текущего пассивного дохода в секунду (IPS). Формула должна учитывать базовое значение дохода от предметов на полке, а также аддитивно применять бонусы от Престижа и Черного рынка, чтобы избежать экспоненциального взрыва в экономике. Учитываем только предметы на полке и базовую аддитивную математику (как MVP).

**Связанная документация (ОПТИМИЗАЦИЯ ТОКЕНОВ):**
- Для вычислений: посмотреть только раздел "1.2 Формула текущего дохода" в файле `.docs/project/03-technical-gdd.md`.
- Для структуры полки: посмотреть `Assets/Scripts/Economy/ShelfManager.cs` (Свойства предмета: `EffectType` и `EffectValue`).
**СТРОГО ЗАПРЕЩЕНО** читать другие базовые файлы из `.docs/project/`, чтобы не раздувать контекст.

**Архитектурные требования:**
1. Файл `IPSCalculator.cs` должен быть размещен в папке `Assets/Scripts/Economy/`.
2. Класс должен содержать статический метод (или метод сервиса):
   `public static double GetCurrentIPS(ShelfManager shelf, double prestigeMultiplier, double blackMarketBonus)`
3. **Логика вычислений:**
   - Шаг 1: `AdjustedBaseIPS`. Суммируем `item.EffectValue` со всех предметов с полки (через `shelf.GetItems()`), у которых `item.EffectType == "Flat_IPS"`.
   - Шаг 2: `Prestige_Bonus = prestigeMultiplier - 1.0;`
   - Шаг 3: `TotalMultiplier = 1.0 + Prestige_Bonus + blackMarketBonus;`
   - Шаг 4: Возвращаем результат `AdjustedBaseIPS * TotalMultiplier;`
   (Внимание: пока что мы не учитываем синергию сетов, реализуем только базовую математику по предметам и бонусам).
4. Оставить метод строгим, математическим. Класс ни от кого не наследуется (если статический), либо наследуется, но не несет в себе UI-логики.
5. Интеграция в LTM: Потребуется создать файл `.docs/systems/IPSCalculator.md` и зарегистрировать его в `INDEX.md`.

**Критерии приемки (Acceptance Criteria):**
- [ ] Разработан класс `IPSCalculator` в папке `Assets/Scripts/Economy/`.
- [ ] Описан метод `GetCurrentIPS`, принимающий нужные параметры `(ShelfManager, double, double)` и возвращающий `double`.
- [ ] Метод корректно суммирует `EffectValue` для `EffectType == "Flat_IPS"`.
- [ ] Мультипликаторы (Престиж и ЧР) складываются **строго аддитивно**, без перемножения друг на друга.
- [ ] Проект компилируется в Unity (нет ошибок).
- [ ] Документация `.docs/systems/IPSCalculator.md` создана (описание функционала и API).
- [ ] Ссылка на документацию системы добавлена в список `.docs/systems/INDEX.md`.
