# Управление UI (UIManager)

**Название:** Базовый синглтон UIManager для UI Toolkit
**Контекст:** Проекту нужен легковесный менеджер для переключения визуальных панелей интерфейса без встраивания туда бизнес-логики. Менеджер нужен для того, чтобы другие системы (например, BootstrapController) могли просто вызывать `UIManager.Instance.Show("ShopPanel")`, а не искать элементы вручную. Вся визуальная часть построена на UI Toolkit.

### Архитектурные требования
1. **Создание файла:** Скрипт `UIManager.cs` должен лежать в `Assets/Scripts/UI/`.
2. **Тип:** Класс должен быть `MonoBehaviour` и реализовывать простой паттерн Singleton (через `public static UIManager Instance { get; private set; }` инициализируемый в `Awake`).
3. **Зависимости:**
   - Ссылка на `UIDocument` (задается через `[SerializeField] private UIDocument document;`).
4. **Кэширование:**
   - После инициализации (в `Start` или `Awake` — удостоверься, что `document.rootVisualElement` доступен) менеджер должен собрать корневые панели, которые мы планируем переключать, в `Dictionary<string, VisualElement>`. 
   - Можно искать по классам (например, `.panel`) или собирать все прямые дочерние элементы корня интерфейса. На данном этапе сделай публичный метод для регистрации панелей или находи их по предопределенному классу (например, все элементы с классом `.screen` или `.panel`). Альтернативно: кэшировать по имени `element.name` любые элементы, которые запрашиваются через API, и сохранять их в Dictionary, чтобы не вызывать `Q()` каждый раз.
5. **Публичное API:**
   - `public void Show(string panelName)`
   - `public void Hide(string panelName)`
   - `public void ShowOnly(string panelName)` (скрывает все закэшированные панели и показывает только запрошенную).
6. **Механизм переключения:**
   - Строго через `VisualElement.AddToClassList("hidden")` и `VisualElement.RemoveFromClassList("hidden")`. 
   - Не использовать `style.display = DisplayStyle.None` напрямую в коде!
7. **Без бизнес-логики:** 
   - UIManager не должен подписываться на кнопки, не должен хранить токены/золото. Он только включает и выключает UI элементы по запросу.
8. **Системный индекс:** Воркер должен задокументировать этот менеджер в `.docs/systems/INDEX.md` (если индекс существует, найти и добавить; если нет — создать раздел UI).

### Критерии приемки (Acceptance Criteria)
- [ ] Файл `UIManager.cs` создан в `/Assets/Scripts/UI/`.
- [ ] Класс реализован как Singleton MonoBehaviour.
- [ ] Реализовано кэширование панелей (`Dictionary<string, VisualElement>`).
- [ ] Присутствуют методы `Show`, `Hide`, `ShowOnly`.
- [ ] Видимость управляется изменениями USS класса `"hidden"`, нет прямых манипуляций `style.display`.
- [ ] В скрипте полностью отсутствует бизнес-логика игры.
- [ ] Запись о `UIManager` добавлена в `.docs/systems/INDEX.md`.
