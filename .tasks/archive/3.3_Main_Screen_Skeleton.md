# M3.3 — Main Screen Skeleton (Visual Only)

**Контекст:**
Цель — собрать единый визуальный каркас (Root UI) с плейсхолдерами для основных элементов экрана (HUD, витрина, кнопка Гачи и навигация) для того, чтобы увидеть всю композицию экрана. 
Это чистая вёрстка без C# логики, в дальнейшем к этим `#id` будут привязываться реальные скрипты и компоненты.

**Связанная документация (ОПТИМИЗАЦИЯ ТОКЕНОВ):**
- Основной референс зон: `.docs/project/09-ui-screens-guidelines.md` (Раздел 1 "Постоянный HUD" и Раздел 2 "Экран Геймплея").
- **Строго запрещено** тратить токены на чтение других базовых архитектурных документов для этой задачи.

**Архитектурные требования (Реализованные):**
1. Создать корневой документ окна-сборщика: `Assets/Scripts/UI_Toolkit/UXML/MainScreen.uxml`.
2. Создать модульные UXML-шаблоны для независимого редактирования и работы с `UIManager`:
   * `HUDPanel.uxml` — Верхняя панель (не скрывается менеджером).
   * `ShelfPanel.uxml` — Игровая зона с 5 витринами (класс `.panel`).
   * `MarketPanel.uxml` — Зона Гачи/Магазина (класс `.panel`).
   * `FooterPanel.uxml` — Нижняя навигация.
3. Создать стили окна: `Assets/Scripts/UI_Toolkit/USS/MainScreen.uss` и подключить в него `Variables.uss` и `Common.uss` (созданные ранее).
4. Верстка выполняется только через **Flexbox** (относительное и абсолютное позиционирование, где уместно для шапки/подвала). Стили подключаются прямо внутрь шаблонов для корректной работы `<ui:Instance>`.
5. **ID элементов:** Каждому логическому узлу, тексту, кнопке назначен уникальный `#id` (например: `#hud-soft-currency`, `#shelf-slot-1`, `#market-spin-btn`, `#nav-btn-upgrades` и т.д.).
6. **Никакой C# логики.** Эта задача только про визуальную сборку (Mockup) структуры. 

**Критерии приемки (Acceptance Criteria):**
- [ ] Окно разбито на 4 независимых файла (`HUDPanel.uxml`, `ShelfPanel.uxml`, `MarketPanel.uxml`, `FooterPanel.uxml`).
- [ ] Шаблоны корректно импортируются и рендерятся внутри `MainScreen.uxml` через `<ui:Instance>`.
- [ ] При открытии UXML в UI Builder структура выглядит завершенной и не ломается (плейсхолдеры занимают свои места).
- [ ] Присутствуют dummy-тексты (например "1.5M") и все нужные элементы из дизайна.
- [ ] Все изменяемые элементы и кнопки имеют `#id`.
- [ ] C# классы и скрипты **НЕ СОЗДАВАЛИСЬ**.
- [ ] (При необходимости) Обновлён `INDEX.md` систем, если добавляется новая структура хранения UXML/USS, хотя для интерфейсов это опционально.
