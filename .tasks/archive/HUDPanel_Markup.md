# Создание верстки HUDPanel (UI Toolkit)

## Контекст
Необходимо сверстать верхнюю панель (шапку), используя макет из Pencil (`gameplay.pen`). Цель — создать 1-к-1 UI Toolkit верстку `HUDPanel.uxml` и обновить стили в `HUD.uss`. Пока без C# логики контроллеров, только чистое UI (Visual Elements).

## Связанная документация
- `.docs/project/09-ui-screens-guidelines.md`
- `.docs/project/02-asset-conventions.md` (раздел по UI Toolkit и именованию)

## Архитектурные требования
- Использовать Flexbox для правильного позиционирования и адаптивности.
- Все классы в `HUD.uss` должны соблюдать BEM или принятые в проекте конвенции(`hud-panel__container` и т.д.).
- Хардкод цветов или размеров в inline-стилях запрещен, всё строго через USS-классы или переменные (`--color-...`).
- Верстка только из стандартных `VisualElement` / `Label` / `Button`.

## Критерии приемки (Acceptance Criteria)
- [ ] Отредактирован/создан файл `HUDPanel.uxml` с использованием правильной семантики (2 логические зоны).
- [ ] Отредактирован/создан файл `HUD.uss`, содержащий все стили панели.
- [ ] К верхней панели применены базовые метрики контейнера: фиксированная высота `137px`, `padding: 32px 0px` (или `0 32px` в зависимости от того, как требует флекс, главное чтобы отступы были равны 32px по нужной оси и 0 по другой), `gap: 16px`.
- [ ] Выстроены 3 логические зоны панели:
    - **Слева:** Баланс (основная софт-валюта) + Токены / Пыль / Ключи.
    - **Справа:** Реф на профиль / Настройки.
- [ ] Шрифты (`Inter`, `Orbitron`) подключены и назначены соответствующим элементам (например, для чисел — Orbitron, для текста — Inter).
- [ ] Применены градиенты или необходимые фоны (в соответствии с макетом `gameplay.pen`).
- [ ] Вызван MCP Pencil для снятия скриншота/сверки с `gameplay.pen` перед и после работы (если необходимо для проверки).
- [ ] Нет ошибок UI Builder, элементы тянутся и сжимаются адекватно.
- [ ] Никаких C# скриптов не заупущено и логика не добавлена (строго View-слой).
