# Название: Создание базовых USS-стилей (Variables и Common)

**Контекст:** Нам необходимо заложить стилистический фундамент интерфейса (сохраняя темную тему и idle-стиль) в обособленных файлах стилей. Это позволит использовать единую палитру, отступы и компоненты во всем проекте без хардкода стилей внутри UXML. 

**Связанная документация (ОПТИМИЗАЦИЯ ТОКЕНОВ):**
- `.docs/project/09-ui-screens-guidelines.md` (Упоминания карточек, редкостей и кнопок).
**Внимание Воркеру:** Строго запрещено читать другие базовые документы GDD. Данная задача — чистая верстка и стилизация!

**Архитектурные требования:**
- USS-файлы должны содержать **ТОЛЬКО** стили (похоже на классический CSS).
- Никакого C#-кода.
- Тематика приложения: Темная тема (Dark theme), idle-игровая эстетика.
- Цвета должны объявляться глобально в `Variables.uss` (через `:root` переменные). `Common.uss` должен использовать эти переменные (с помощью `var(--имя-переменной)`).

**Критерии приемки (Acceptance Criteria):**
- [ ] Вручную создана директория для USS (если её нет): `Assets/Scripts/UI_Toolkit/USS/`.
- [ ] Файл `Assets/Scripts/UI_Toolkit/USS/Variables.uss` создан и в нём заведены CSS-переменные для тёмной idle-темы (primary, secondary/background цвета, размеры шрифтов, `border-radius`, стандартные отступы).
- [ ] В `Variables.uss` также заведены цветовые переменные для редкостей (Rare, Epic, Legendary, Unique).
- [ ] Файл `Assets/Scripts/UI_Toolkit/USS/Common.uss` создан и содержит базовые утилитарные классы: `.hidden` (обязательно с `display: none;`), `.btn-primary`, `.card`, `.panel`.
- [ ] В `Common.uss` реализованы классы для цветовых аур ("свечения" или рамки) редкостей: `.rarity-rare`, `.rarity-epic`, `.rarity-legendary`, `.rarity-unique`.
- [ ] Оба файла не содержат синтаксических ошибок и готовы к подключению в UI Builder.
