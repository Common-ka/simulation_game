# 1.3 — GameManager: Игровой тик-контроллер

## Название
Создать `GameManager.cs` — главный тик-контроллер сцены `Game.unity`

---

## Контекст
`GameManager` — сердце игрового цикла. Он запускает периодический тик раз в секунду (`InvokeRepeating`) и уведомляет всех подписчиков об изменении игрового состояния через событие `OnGameStateChanged`. На этом этапе реализуется минимальная версия (M1): просто прибавлять `softCurrency += 1` и публиковать событие. Сложная экономика (IPS, апгрейды) подключается позже.

## Архитектурные требования

1. **Расположение файла:** `Assets/Scripts/Core/GameManager.cs`
2. **MonoBehaviour** с атрибутом `[DisallowMultipleComponent]`
3. **Тик:** `InvokeRepeating(nameof(GameTick), 0f, 1f)` в `Start()`
4. **Правило изоляции:** `GameManager` не знает про UI. Никаких `UIDocument`, `Label`, `VisualElement` и т.п.
5. **Публичное событие:** `public static event Action<GameSnapshot> OnGameStateChanged` — статическое, чтобы UI-панели могли подписаться без прямой ссылки на объект.
6. **Снапшот:** создать простую структуру `GameSnapshot` (в том же файле или рядом) с полем `double SoftCurrency`.
7. **Запрет Magic Numbers:** тик-интервал (`1f`) — вынести в `[SerializeField] private float _tickInterval = 1f`.
8. **Нет `GetComponent` в Update/Tick.** Метод `GameTick()` только мутирует данные и вызывает событие.
9. **Нет LINQ, нет аллокаций** внутри `GameTick()`.

## Технические детали реализации

```csharp
// Минимальный скелет для ориентира:
public class GameManager : MonoBehaviour
{
    [SerializeField] private float _tickInterval = 1f;

    private double _softCurrency;

    public static event Action<GameSnapshot> OnGameStateChanged;

    private void Start()
    {
        InvokeRepeating(nameof(GameTick), 0f, _tickInterval);
    }

    private void GameTick()
    {
        _softCurrency += 1;
        OnGameStateChanged?.Invoke(new GameSnapshot { SoftCurrency = _softCurrency });
    }
}

public struct GameSnapshot
{
    public double SoftCurrency;
}
```

> ⚠️ Это ориентир. Воркер может улучшить структуру, но не должен нарушать правила выше.
