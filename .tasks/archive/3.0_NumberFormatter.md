# ТЗ: NumberFormatter.cs — Утилитный класс для форматирования чисел

## Контекст
Необходимо создать статический класс-утилиту для удобного форматирования больших чисел (от 1000 до квадриллионов). Это нужно для того, чтобы цифры (баланс, цены) в UI были легко читаемыми (например, 123.4K вместо 123400).

## Связанная документация (ОПТИМИЗАЦИЯ ТОКЕНОВ)
- **Строго запрещено** тратить токены на чтение общих архитектурных файлов (UI, GDD и т.д.), так как задача полностью изолирована.
- Требуется только обновить индекс систем: `.docs/systems/INDEX.md`.

## Архитектурные требования
- **Расположение:** Файл должен быть создан по пути `Assets/Scripts/Utils/NumberFormatter.cs`. Если папки `Utils` нет — создай её.
- **Тип:** Класс должен быть `public static` (без `MonoBehaviour`).
- **Зависимости:** Полное отсутствие внешних зависимостей. Только чистый C# (`System`).
- **Сигнатура метода:** `public static string Format(double value)`
- **Правила форматирования:**
  - `< 1000` → оригинальное число (например, `123` или `999`).
  - `< 1,000,000` → `K` (например, `123.4K`).
  - `< 1,000,000,000` → `M` (например, `123.4M`).
  - `< 1,000,000,000,000` → `B` (например, `1.2B`).
  - `< 1,000,000,000,000,000` → `T` (триллионы).
  - `>= 1,000,000,000,000,000` → `Q` (квадриллионы).
  - Число должно иметь **не более одного знака после запятой** (используй `0.#` формат, чтобы "120.0" превращалось в "120", но "123.4" оставалось "123.4", либо `F1` по усмотрению, главное чтобы выглядело аккуратно).
