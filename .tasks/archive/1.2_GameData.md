# Задача 1.2 — GameData: Загрузка локальных конфигов

## Название
GameData: Написать `GameDataLoader.cs` для чтения готовых JSON из `Resources/GameData/`.

---

## Контекст
Все JSON-файлы уже существуют в `Assets/Resources/GameData/`. Папка уже создана.
`BootstrapController` (задача 1.1) вызывает `GameDataLoader` на шаге 1 инициализации — нужно реализовать сам загрузчик.

**Ограничения:**
- Только **локальное** чтение через `Resources.Load<TextAsset>()`.
- **Запрещено** читать `StreamingAssets/remote_config.json` и использовать `UnityWebRequest` — это задача будущего M8.
- **Запрещено** делать JSON-парсинг вне `GameDataLoader`.

## Существующие JSON-файлы и их структура

Перед написанием кода **прочитай схемы** этих файлов через `read_resource` или `view_file`:

| Файл | Тип | Примечание |
|---|---|---|
| `Categories.json` | Массив объектов `{ Name, UnlockCost }` | `UnlockCost` — большие числа (до 2e20), использовать `double` |
| `GachaMath.json` | Объект с вложенными блоками | Словари редкостей — использовать `[Serializable]` классы |
| `BlackMarketConfig.json` | Объект | Прочитать перед написанием модели |
| `Upgrades.json` | Массив | Прочитать перед написанием модели |

> `LootTable.json` (141 KB) и `BlackMarketArtifacts.json` — **не грузить в этой задаче**. Они будут подключены в задачах `EconomyService` и `BlackMarketManager`.

## Что нужно создать

### `GameDataLoader.cs`
**Путь:** `Assets/Scripts/Economy/GameDataLoader.cs`

**Требования:**
- `MonoBehaviour` (Bootstrap держит ссылку, не синглтон).
- Публичный метод `LoadAsync()` → `IEnumerator` (вызывается через `StartCoroutine` из `BootstrapController`).
- Внутри `LoadAsync()` загрузить **4 файла**: `Categories`, `GachaMath`, `BlackMarketConfig`, `Upgrades`.
- При отсутствии файла — `Debug.LogError(...)`, корутина продолжает работу (не крашится).
- Парсинг через **`Newtonsoft.Json`** (`JsonConvert.DeserializeObject<T>()`) — пакет установлен в проекте.
- Публичные свойства только для чтения после загрузки.

> ⚠️ `UnlockCost` в `Categories.json` достигает `2e20` — тип поля должен быть `double`, не `long` и не `int`.

## Подключение в `BootstrapController`

В `BootstrapController.cs` уже есть шаг загрузки данных. Воркер должен:
1. Добавить `[SerializeField] private GameDataLoader _gameDataLoader;`.
2. Заменить заглушку на `yield return StartCoroutine(_gameDataLoader.LoadAsync());`.
3. Назначить компонент через инспектор (или `GetComponent` в `Awake`, если на том же GameObject).
