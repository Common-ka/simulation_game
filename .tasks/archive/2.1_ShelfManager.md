# ТЗ: 2.1_ShelfManager.md

- **Название:** Менеджер полок (ShelfManager)
- **Контекст:** Необходимо реализовать ядро физики экономики — систему полок. Полка служит источником пассивного дохода в игре и имеет ограниченное количество слотов (5 штук).
- **Связанная документация (ОПТИМИЗАЦИЯ ТОКЕНОВ):** 
  - `.docs/project/05-macro-balance.md` (Раздел 3: Полки и маршрутизация статов)
  - Не читай другие архитектурные документы, кроме базовых утилит, если это не требуется напрямую.
- **Архитектурные требования:**
  - `ShelfManager.cs` должен быть размещен в папке `Assets/Scripts/Economy/`.
  - **Без UI-связей**: ShelfManager не знает про UI-компоненты, он оперирует только данными (Event-driven архитектура).
  - Исключить любые тяжелые вычисления в `Update()`.
- **Критерии приемки (Acceptance Criteria):**
  1. [ ] Класс поддерживает строго 5 слотов под предметы.
  2. [ ] Логика добавления предмета Rare+: если полка не заполнена — ставим сразу. Вся остальная сложная логика вытеснения отменена/отсутствует в базовом макро-балансе.
  3. [ ] При любом изменении состояния полок бросается `event` или `Action` -> `OnShelfUpdated`.
  4. [ ] Новая система (ShelfManager) зарегистрирована разработчиком в LTM через `.docs/systems/INDEX.md` после реализации.
