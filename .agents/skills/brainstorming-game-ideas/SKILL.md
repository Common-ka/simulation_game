# Game Design Brainstorming Engine

## Когда использовать этот скилл
- Пользователь просит придумать новые геймплейные фичи (кор-луп, мета, эндгейм).
- Возникла проблема с вовлечением (retention) игроков или экономикой, и нужны идеи решения.
- Нужна генерация тематик для Live-Ops (ивентов, акций, скинов).

## Чеклист (Workflow)
- [ ] 1. Изучить контекст запроса.
- [ ] 2. **Дивергенция:** Мозговой штурм (генерация 5-10 максимально разных, нестандартных идей, используя фреймворк ниже).
- [ ] 3. **Конвергенция:** Жесткая фильтрация идей (оставить только топ-3, отбросив нереализуемый мусор).
- [ ] 4. Вывести финальный результат строго в виде форматированной таблицы.

## Инструкции

### Фаза 1: Генерация (Эвристика SCAMPER)
Не предлагай банальности (ежедневные квесты, баттлпасс, рулетка) без уникальной "киллер-фичи". Используй метод SCAMPER для модификации старых идей:
- **Substitute (Заменить):** Что мы можем заменить? (Напр. вместо накопления энергии — накопление стресса).
- **Combine (Скомбинировать):** Как соединить две наши текущие разрозненные механики в одну мощную синергию?
- **Reverse (Поменять местами):** Что если нарушить правила жанра? (Напр. игрок не зарабатывает деньги, а должен тратить их как можно быстрее во избежание дефолта).

### Фаза 2: Валидация (Фильтры жизнеспособности)
Прежде чем выдавать идею в ответ, мысленно прогони её через три фильтра. Идеи, не прошедшие проверку, **УНИЧТОЖИТЬ**:
1. **Код / Unity Architecture:** Можно ли это запрограммировать без тотального рефакторинга и переписывания ядра (Core Simulation)?
2. **Баланс / Idle Economy:** Не сломает ли фича инфляцию? (Например, идеи, генерирующие бесконечные ресурсы без сжигания / Resource Sinks — удаляются).
3. **Монетизация & Удержание:** Есть ли в фиче органичная точка боли, чтобы игрок захотел посмотреть рекламу (Rewarded) или купить IAP?

### Фаза 3: Шаблон вывода
Выводи результаты строго структурно. Избегай длинных параграфов воды.

```markdown
### 🧠 Штурм Механик

| Название Идеи | Core Loop (В чем коротко суть) | Code Risk (Оценка кодинга) | Оценка Геймдизайна (Fun Factor) |
| :--- | :--- | :--- | :--- |
| **Название 1** | Выжимка механики | 🟢 Low / 🟡 Med / 🔴 High | Почему это затянет игрока |
| **Название 2** | Выжимка механики | 🟢 Low / 🟡 Med / 🔴 High | Почему это затянет игрока |

### 🛠 Разбор Финалиста (Лучшая идея)
**[Название идеи-победителя]**
- **Как это впишется в экономику:** (Пример влияния на инфляцию).
- **Как будем это монетизировать:** (Пример интеграции Rewarded AD).
- **Архитектурный шаг:** (С чего технически начнет писать код программист).
```
