# Antigravity Skill Creator

## Когда использовать этот скилл
- Нужно заархивировать успешный код / опыт решения задачи.
- Ты выступаешь в роли Архивариуса и должен создать файл `SKILL.md` для будущих Воркеров.

## Чеклист (Workflow)
- [ ] 1. Изучить контекст задачи или решения, из которого собирается скилл.
- [ ] 2. Сгенерировать название в формате Gerund (например, `testing-code`, `creating-ui`).
- [ ] 3. Написать описание (description) от 3-го лица с триггером.
- [ ] 4. Вычленить воду, оставив только шаги и шаблоны кода.
- [ ] 5. Создать файл `SKILL.md` через инструмент `write_to_file`.

## Инструкции

### 1. Архитектура папок
Каждый скилл должен лежать по пути `.agents/skills/<skill-name>/`.
В папке скилла кроме `SKILL.md` (обязательно) могут лежать `scripts/`, `examples/` или `resources/` (опционально, если код большой).

### 2. YAML Frontmatter (ОБЯЗАТЕЛЬНО)
Файл обязан начинаться с метаданных (шапки). Строгие правила:
- **name**: Форма Gerund (например, `managing-databases`). Поддерживаются только строчные английские буквы, цифры и дефисы. Никаких слов `claude` или `anthropic`. Макс. 64 символа.
- **description**: Пишется от 3-го лица. Строго содержит триггеры вызова (например: "Используй этот скилл, когда пользователь просит..."). Макс. 1024 символа.

Пример:
```yaml
---
name: format-optimization
description: Оптимизирует форматы данных. Вызывается, когда необходимо сжать данные.
---
```

### 3. Правила написания (Claude Way)
- **Лаконичность:** Агенты умные. Не объясняй им, что такое Git. Описывай только специфичную логику.
- **Плавное погружение (Progressive Disclosure):** Текст скилла не должен превышать 500 строк. Для больших деталей используй ссылки на другие файлы (только 1 уровень вложенности).
- **Слеши:** Только `/` для путей, никогда не используй `\`.
- **Свобода действий:**
   - Используй **Bullet Points** для эвристики (если агент может подумать сам).
   - Используй **Code Blocks** для кусков кода.
   - Используй **Specific Bash Commands** для хрупких или точных системных операций.

### 4. Сложные флоу
- Добавляй **Чеклисты** (Checklists), чтобы агент мог скопировать их в свой Task.md и отслеживать прогресс.
- Описывай валидацию: "Сначала запусти скрипт валидации X, потом меняй конфиг".

### 5. Шаблон тела файла SKILL.md
```markdown
# [Название Скилла]

## Когда использовать этот скилл
- [Триггер 1]
- [Триггер 2]

## Чеклист (Workflow)
[Маркдаун чеклист шагов агента]

## Инструкции
[Логика, фрагменты кода]

## Ресурсы (опционально)
- [Ссылки на вспомогательные файлы]
```
